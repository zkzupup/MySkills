# Document Output Conventions

> Reference file for file placement, naming, and cross-referencing. Consult this when creating or organizing output files.
> For analysis methodology, see [SKILL.md](SKILL.md). For document templates, see [templates.md](templates.md).

## Document Output Structure

All generated documentation MUST follow this directory layout. The goal is strict **separation of concerns**: narrative analysis, diagrams, and presentation materials each have their own home.

### Top-Level Layout

```
Document/
├── MasterIndex.md                 # Master index -- the single navigation hub
├── <ConventionDocs>.md            # Convention / standard docs (optional, project-specific)
│
├── NN-<TopicName>/                # Numbered topic folders (analysis markdown)
│   ├── <Module>_Analysis.md
│   └── ...
│
├── diagrams/                      # ALL Excalidraw diagrams (grouped by domain)
│   ├── <DomainLabel>/
│   │   ├── *.excalidraw           # Editable source files
│   │   └── *.png                  # Exported viewable files (generated by export script)
│   └── ...
│
└── Presentation/                  # (Optional) Presentation-oriented materials
    ├── *.md
    └── diagrams/
        └── Dnn-<Description>.excalidraw  # Presentation-specific diagrams with order prefix
```

### Directory Naming Rules

| Level | Pattern | Example |
|-------|---------|---------|
| Topic folder | `NN-<TopicName>` (two-digit zero-padded prefix) | `01-Overview`, `02-CoreModuleA` |
| Diagram subfolder | Short domain label (no number prefix) | `CoreModuleA`, `Infrastructure` |
| Presentation diagram | `Dnn-<Description>` or `nn-<Description>` | `D01-SystemOverview`, `03-DataFlow` |

**Topic folder numbering** reflects reading order: general overview first, then subsystems by importance, finally utilities/tools.

### File Naming Rules

| File Type | Pattern | Example |
|-----------|---------|---------|
| Module analysis doc | `<ModuleName>_Analysis.md` | `Auth_Framework_Analysis.md` |
| End-to-end flow doc | `<FlowName>_E2EFlow.md` | `OrderFlow_E2EFlow.md` |
| Specialized analysis | `<Topic>_Analysis.md` | `CacheEviction_Analysis.md` |
| Architecture diagram | `<SystemName>_Architecture.excalidraw` | `Payment_Architecture.excalidraw` |
| Flow/pipeline diagram | `<Description>_Pipeline.excalidraw` | `DataIngestion_Pipeline.excalidraw` |
| Lifecycle diagram | `<Object>_Lifecycle.excalidraw` | `Session_Lifecycle.excalidraw` |
| State machine diagram | `<Object>_StateMachine.excalidraw` | `Order_StateMachine.excalidraw` |
| Class/relation diagram | `<Object>_ClassDiagram.excalidraw` | `Entity_ClassDiagram.excalidraw` |
| Memory layout diagram | `<Object>_MemoryLayout.excalidraw` | `Bucket_MemoryLayout.excalidraw` |
| Exported PNG (any type) | `<same-base-name>.png` | `Payment_Architecture.png` |

**General principles:**
- Domain-specific English terms (class names, technical abbreviations) keep their original form
- Use the project's primary language for descriptive text; preserve technical terms as-is
- No spaces in file names; use underscores for English compound words
- File extension determines type: `.md` for docs, `.excalidraw` for diagrams
- Choose a consistent naming language (Chinese or English) for the project and stick with it

### Diagram Subfolder Alignment

Each `NN-<TopicName>` topic folder MUST have a corresponding `diagrams/<ShortLabel>/` subfolder. Decide the short labels during Phase 1 when the module inventory is built.

**Rules:**
- The short label should be a concise domain keyword derived from the topic folder name
- Overview-level diagrams that span multiple modules can live at `diagrams/` root
- Cross-cutting or miscellaneous diagrams go into a catch-all subfolder (e.g., `diagrams/Misc/`)
- When a new topic folder is added, its corresponding diagram subfolder MUST be created immediately (even if initially empty)

### Cross-Referencing Convention

- Analysis docs reference diagrams using **relative paths** to the `.png` export: `![Architecture](../diagrams/<SubsystemLabel>/<DiagramName>.png)`
- The master index lists ALL docs and ALL diagrams with relative links
- Every analysis doc MUST be listed in the master index; orphan docs are prohibited

### Adding New Topic Folders

When the project scope expands:

1. Assign the next sequential number: `NN-<NewTopicName>`
2. Create the corresponding `diagrams/<ShortLabel>/` subfolder
3. Register in master index immediately
4. Follow all naming conventions above

## Diagram Strategy

Select the diagram workflow based on whether the `excalidraw-workflow` skill is available.

### Path A: With `excalidraw-workflow` (Recommended)

1. **Generate**: Following the Phase 1 diagram plan, invoke the `excalidraw-workflow` skill to produce `.excalidraw` files into `diagrams/<DomainLabel>/`
2. **Export to PNG**: Run the export command immediately after each batch of diagrams:
   ```bash
   node <excalidraw-workflow-skill-root>/scripts/export_excalidraw_png.js ./Document/diagrams
   ```
   Incremental by default (skips already-exported files). Use `--force` to re-export after edits.
3. **Reference rule**: All `![...]()` embeds in documents MUST point to `.png`, never `.excalidraw`
4. **Index registration**: The Master Index diagrams table lists the `.png` path for viewing and notes the `.excalidraw` source exists for editing
5. **Export timing**: Export immediately after Phase 2 global architecture diagram; batch-export after all Phase 3 diagrams per module; run once more on the full `diagrams/` directory before writing handoff notes

### Path B: Without `excalidraw-workflow`

1. **Generate**: Write **detailed Mermaid diagrams** directly in the analysis document (use box-drawing characters, label component names and connections)
2. **Placement**: Mermaid diagrams are inline within the diagram sections of analysis documents; no separate files needed
3. **Index marking**: Mark the module's diagram status as `[mermaid-only]` in the Master Index
4. **Upgrade path**: When `excalidraw-workflow` becomes available later, Mermaid diagrams can be batch-converted to Excalidraw format

## Structure Compliance Checklist

Run this checklist at the end of each conversation to verify the Document directory remains well-structured.

### Directory Rules
- [ ] All topic folders follow `NN-<TopicName>` naming (two-digit, zero-padded)
- [ ] Topic folder numbers are sequential with no gaps
- [ ] `diagrams/` subfolders exist for every topic domain
- [ ] No `.excalidraw` files outside `diagrams/` or `Presentation/diagrams/`
- [ ] No `.md` analysis files outside their `NN-` topic folder
- [ ] Root level contains only index/convention files, not analysis docs

### File Naming Rules
- [ ] Analysis docs use descriptive name + analysis suffix + `.md`
- [ ] Diagram files use descriptive name + type suffix + `.excalidraw`
- [ ] No spaces in file names
- [ ] Domain-specific technical terms preserved in original form

### Cross-Reference Integrity
- [ ] Master index lists ALL analysis docs with working relative links
- [ ] Master index lists ALL diagrams with working relative links
- [ ] Analysis docs reference diagrams using relative paths (`../diagrams/...`)
- [ ] No orphan files (every file registered in master index)
- [ ] No broken links in master index
- [ ] Every `.excalidraw` file under `diagrams/` has a corresponding `.png` sibling
- [ ] All `![...]()` image embeds in analysis docs reference `.png` files, not `.excalidraw`
